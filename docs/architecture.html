Factory Inventory Management System

Architecture overview — a full-stack demo application with a Vue 3 single-page frontend, a Python FastAPI backend, and in-memory mock data loaded from JSON files. No database.

Vue 3 + Vite FastAPI In-memory JSON data REST API pytest

System Architecture

Three layers: browser SPA on port 3000, REST API on port 8001, and JSON files loaded into memory at server startup.

Tech Stack

Deliberately minimal — three runtime dependencies on the frontend, FastAPI on the backend, JSON files instead of a database.

Frontend

  • Vue 3 (Composition API) ^3.4
  • vue-router ^4.3
  • axios ^1.6
  • Vite dev server / build ^5.2
  • Custom SVG charts no chart lib

Backend

  • Python + FastAPI uv-managed
  • Pydantic response models
  • CORS middleware allow all (dev)
  • Uvicorn port 8001
  • No auth, no rate limiting demo only

Data & Testing

  • 7 JSON files in server/data/
  • Loaded once at startup, held in memory
  • No persistence — restart reloads
  • pytest + FastAPI TestClient tests/backend
  • generate_data.py for sample data

Key Design Decisions

Global filter system

Four filters (Time Period, Warehouse, Category, Order Status) live in the useFilters composable as module-level refs, so every view shares the same state. Views watch the refs and re-fetch; the backend does all filtering server-side. Inventory has no time dimension, so month filters don't apply to it.

In-memory data, no database

All datasets load from JSON once at import time and are filtered with plain Python list comprehensions per request. Nothing persists — restarting the server resets state. This keeps the demo dependency-free and fast for <10K records.

Reactivity pattern

Raw API responses land in refs (allOrders, inventoryItems); everything derived — chart series, KPI totals, filtered tables — is a computed property. Charts are hand-built SVG, no charting library.

Typed API contract

Every list endpoint declares a Pydantic response_model, so the JSON data files must stay in sync with the models — changing a JSON field without updating the model is the most common way to break the API.
